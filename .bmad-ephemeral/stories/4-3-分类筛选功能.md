# Story 4.3: 分类筛选功能

Status: done

## Story

As a **reader**,  
I want **to filter articles by category**,  
So that **I can find articles on specific topics**.

## Acceptance Criteria

Based on Epic 4 Story 4.3 from epics.md and tech-spec-epic-4.md:

1. **AC-4.3.1:** Given I am on the homepage or article list, When I click on a category, Then I see only articles in that category
2. **AC-4.3.2:** Given I am viewing a filtered category page, Then the URL reflects the filter (e.g., /articles/category/技术)
3. **AC-4.3.3:** Given I am viewing a category page, Then I can see how many articles are in this category
4. **AC-4.3.4:** Given I am viewing a filtered category page, When I click "All" or remove the filter, Then I see all articles again
5. **AC-4.3.5:** Given I am viewing a category page, Then the page displays the category name and description (if available)
6. **AC-4.3.6:** Given I am viewing a category page with no articles, Then I see a friendly empty state message
7. **AC-4.3.7:** Given I access a category that doesn't exist, Then I see a 404 error page

## Dev Notes

### Relevant Architecture Patterns and Constraints

**Server Components:**
- Use Next.js Server Components for category page to enable SSR and SEO
- Fetch data directly in Server Component using Prisma (more efficient than API calls)
- Use `async/await` for asynchronous data fetching
- Reference: [Source: docs/architecture.md#Next.js-App-Router]

**API Integration:**
- Public API endpoint available: `GET /api/articles/public?categorySlug=[slug]`
- API returns unified response format: `{ success: boolean, data: T, error?: { message: string, code: string } }`
- API handles category filtering by slug (converts slug to categoryId)
- Reference: [Source: app/api/articles/public/route.ts:50-76]

**Component Organization:**
- Article components in `components/article/` directory
- Follow naming convention: `PascalCase.tsx` for components
- Reuse existing components: ArticleList, ArticleCard, Pagination
- Reference: [Source: docs/architecture.md#Component-Organization]

**Styling:**
- Use Tailwind CSS for all styling
- Mobile-first responsive design
- Use Tailwind responsive breakpoints: `sm:`, `md:`, `lg:`, `xl:`
- Reference: [Source: docs/architecture.md#Styling-Architecture]

**Navigation:**
- Use Next.js Link component for client-side navigation
- Category links should navigate to `/articles/category/[slug]`
- "All" link should navigate to homepage (`/`)
- Reference: [Source: docs/architecture.md#Navigation-Patterns]

**Error Handling:**
- Use unified error format
- Display user-friendly error messages
- Log errors for debugging (console in dev, structured in prod)
- Use Next.js `notFound()` for 404 errors
- Reference: [Source: docs/architecture.md#Error-Handling]

**SEO:**
- Use Next.js Metadata API for dynamic meta tags
- Add title and description based on category
- Reference: [Source: .bmad-ephemeral/stories/tech-spec-epic-4.md#Story-43-分类筛选功能]
