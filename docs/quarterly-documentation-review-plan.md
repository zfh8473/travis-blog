## 📊 审查流程

### Step 1: 准备阶段（审查前 1 周）

1. **通知团队**
   - 发送审查通知
   - 说明审查范围和时间

2. **收集信息**
   - 收集本季度所有代码变更
   - 收集本季度所有功能变更
   - 收集技术栈变更记录

3. **准备检查清单**
   - 基于变更记录准备检查项
   - 准备审查工具

---

### Step 2: 执行审查（审查周）

1. **执行自动检查**
   ```bash
   npm run docs:check
   ```

2. **手动审查文档**
   - 逐项检查检查清单
   - 对比文档与实际实现
   - 记录发现的问题

3. **生成审查报告**
   - 记录发现的问题
   - 记录需要更新的文档
   - 记录更新优先级

---

### Step 3: 更新阶段（审查后 1 周）

1. **执行文档更新**
   - 按照优先级更新文档
   - 记录更新内容
   - 提交文档更新

2. **审查更新**
   - 审查更新后的文档
   - 确认问题已解决
   - 更新审查报告

---

### Step 4: 总结阶段（审查后 2 周）

1. **生成审查总结**
   - 总结发现的问题
   - 总结更新内容
   - 总结改进建议

2. **团队分享**
   - 分享审查结果
   - 分享改进建议
   - 更新文档维护流程（如需要）

## 📈 审查指标

### 关键指标

1. **文档一致性率**
   - 目标：≥ 95%
   - 计算：一致文档数 / 总文档数 × 100%

2. **问题发现数**
   - 记录每次审查发现的问题数
   - 跟踪问题趋势

3. **更新及时性**
   - 目标：审查后 1 周内完成更新
   - 测量：审查到更新的时间
