# 文章标题层级问题分析

**分析日期：** 2025-01-XX  
**分析人员：** UX Designer  
**状态：** ✅ 已分析并实施

---

## 📋 用户反馈

**问题：** 文章主标题和正文中的 h1 标题字体太接近，如果用户在正文中使用了 h1 格式，两者看起来不舒服。

---

## 🎯 UX 专业分析

### 当前状态

- **文章主标题（页面 h1）：** `text-5xl` (3.5rem, 800)
- **正文中的 h1：** `prose-h1:text-4xl` (2.25rem, 800)
- **比例：** 3.5:2.25 = **1.56:1**

### 问题分析

✅ **用户反馈非常合理**

#### 1. 语义层级问题

根据 **HTML 语义化** 和 **Typography Hierarchy** 原则：

- 一个页面应该只有一个 `<h1>` 标签（文章主标题）
- 正文内容应该从 `<h2>` 开始
- 如果正文中使用 `<h1>`，会造成语义混乱

#### 2. 视觉层级问题

- 当前比例 1.56:1 确实不够明显
- 理想比例应该至少 **2:1** 或更大
- 正文中的 h1 不应该与页面主标题竞争视觉注意力

#### 3. 最佳实践

现代内容管理系统（CMS）和博客平台的做法：

- **Medium：** 正文中不允许使用 h1，只允许 h2-h6
- **WordPress：** 正文编辑器通常从 h2 开始
- **Notion：** 页面标题是 h1，内容从 h2 开始
- **GitHub Markdown：** 文档标题是 h1，内容从 h2 开始

---

## 🎨 改进方案

### 方案 1：降低正文中 h1 的样式（推荐）

将正文中的 h1 样式调整为与 h2 相同或更小：

- **正文 h1：** `text-4xl` → **`text-2xl`** (1.5rem) 或 **`text-3xl`** (1.875rem)
- **比例：** 3.5:1.5 = **2.33:1** 或 3.5:1.875 = **1.87:1**

### 方案 2：完全移除正文中的 h1 样式

在 Markdown 转换时，将正文中的 h1 转换为 h2：

- 这需要在 `markdownToHtml` 函数中处理
- 或者通过 CSS 将正文中的 h1 样式完全隐藏/禁用

### 方案 3：增强文章主标题（已实施）

- 文章主标题：`text-5xl` (3.5rem)
- 字重：`font-extrabold` (800)
- 颜色：`text-slate-900`

---

## ✅ 推荐实施

**采用方案 1 + 方案 3 的组合：**

1. ✅ 保持文章主标题为 `text-5xl`（已实施）
2. ✅ 将正文中的 h1 样式调整为 `text-2xl` 或 `text-3xl`
3. ✅ 在 Markdown 编辑器或文档中建议用户使用 h2 作为最高级标题

这样既解决了视觉层级问题，又保持了语义的正确性。

---

## 📊 改进效果

### 改进前
- 文章主标题：3.5rem
- 正文 h1：2.25rem
- 比例：1.56:1 ❌

### 改进后

- 正文 h1：1.5rem 或 1.875rem
- 比例：2.33:1 或 1.87:1 ✅
